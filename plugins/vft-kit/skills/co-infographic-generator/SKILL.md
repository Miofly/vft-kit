# 信息图生成器（HTML + puppeteer）

## 它解决什么问题

把一段结构化内容（表格、要点、流程、成果数字）变成一张**风格统一、中文准确、高大上**的 PNG 图片。

为什么是 HTML/CSS 而不是 Mermaid 或文生图模型：
- **文生图模型**（如 Z-Image）渲染中文会糊字、错字，且不可编辑——做信息图不可靠。
- **Mermaid** 做不出渐变、玻璃拟态、精确网格、序号徽标这类质感。
- **HTML/CSS** 给你像素级控制 + 100% 准确的中文 + 随时改文字重渲——这是本 skill 的核心选择。

渲染靠 puppeteer 截图。**依赖自给自足**：首次运行若本机没有 puppeteer，脚本会自动装到 `~/.cache/vft-kit/render`（只装 JS 包，秒级），浏览器则复用机器上已有的 chromium 或系统 Chrome/Edge，**不会下载 ~110MB 的浏览器**。装一次，之后所有项目复用。

## 工作流

### 0. 先对齐「风格 + 版式」（除非用户已说明）
信息图的观感八成由**配色风格**和**版式**决定，而这两件事很主观——同一份内容，科技青和暖金高端给人的感觉天差地别。所以除非用户已经给了明确信号（如「暖色调」「做成对比图」「暗黑科技风」），否则**先用一两句话快速确认，别默认套青色科技风就开干**。问法保持轻量，给选项而不是开放式审问，例如：

- **配色 / 气质**：科技青（默认）· 暖金高端 · 靛紫品红 · 翠绿青柠 · 暗黑酷感，或用户自定义主色。配方表见 `references/design-system.md`，换色主要改 `--accent` / `--accent2`（记得连背景光晕一起换）。
- **版式倾向**：并列卡片 · 指标看板 · 横向流程 · 纵向时间线 · A/B 对比，或几种组合 / 自定义排布。

判断尺度：信息**已经够了就直接做**，别为走流程而追问；**有明显多解、又没线索时才问**。这一步是为了少返工，不是仪式。

### 1. 定版式：模板是「起点」，不是「菜单」
`assets/templates/` 里有 5 个开箱即用的布局，覆盖最常见的内容形态，是「改文字就能用」的完整 HTML：

| 内容形态 | 起手模板 |
|----------|----------|
| 并列的几条要点 / 前置条件 / 特性 | `card-grid.html` |
| 一组数字成果 / 指标看板 | `kpi-dashboard.html` |
| 主流程步骤（只要标题） | `step-flow.html` |
| 流程大纲（含每步要点/子项） | `timeline.html` |
| A vs B / 前后对比 | `comparison.html` |

**但别被这 5 个框死。** 真正让多张图「成系列、显专业」的是 `design-system.md` 里那套共享 CSS token 和组件（卡片、序号徽标、标签芯片、对比胶囊、进度条、渐变大数字、线性图标），**不是某个具体模板**。当内容不能干净地落进某一个模板时，优先**用这些组件自由组合一个新版式**，而不是硬把内容塞进最接近的模板：

- 内容有多个层次 → 把看板 + 对比、流程 + 要点**拼在一张图里**分区呈现；
- 模板的列数 / 结构不合适 → 直接改 HTML 结构，只要沿用同一套 token，风格依然统一；
- 全新形态（矩阵、四象限、中心辐射、雷达式罗列…）→ 从 token 起手新写一个，design-system.md 的组件就是你的积木。

拿不准时就近选一个起手模板，再按内容裁剪——模板是脚手架，不是牢笼。

### 2. 写 HTML
把选中 / 组合好的版式落到工作目录（建议项目下 `charts-output/`），替换占位文字、增删条目（每个模板顶部注释说明了怎么加）。要点：
- 整张图必须包在 `id="shot"` 容器里（渲染脚本截这个元素）。
- `body` 字体保留 `"Microsoft YaHei","PingFang SC"`，否则中文会缺字。
- 用第 0 步定好的配方：换主题色改 `--accent` / `--accent2`，**并把 `#shot` 背景光晕的颜色同步成新主色**（否则暖色配青色光晕会不搭，详见 design-system.md）；要暗黑科技风用其「暗色变体」。
- 图标从 `assets/icons.md` 取线性 SVG，**不要用 emoji**。
- 关键词用 `<b>` 加粗，提升信息层次。

设计细节、CSS token、组件样式、配色配方全部在 `references/design-system.md`，动手前先读它。

### 3. 渲染成 PNG
```bash
node <skill>/scripts/render.mjs <input.html> <output.png> [--scale 3] [--no-install]
```
- 默认截 `#shot` 元素、3 倍分辨率高清；找不到 `#shot` 时告警并退化为整页截图。
- **图宽由 HTML 里 `#shot { width }` 决定，没有命令行宽度参数**——要改宽度就改 HTML 重渲。
- 首次运行会自动装 puppeteer（见上文），`--no-install` 可禁止自动安装（CI 里想显式控制依赖时用）。
- 截图前会自动扫描横向溢出，溢出会打印超出多少 px 及是哪个元素。

浏览器实在找不到时（罕见），下载一次：`npx --prefix ~/.cache/vft-kit/render puppeteer browsers install chrome`。

### 4. 自检并交付
渲染脚本已自动检查内容溢出（有告警就按提示调宽 `#shot` 重渲）。剩下的用 Read 工具看一眼生成的 PNG 确认：
- 中文无缺字/方块、无错字；
- 风格与同文档其它图一致。

把 PNG 路径交给用户，并保留对应 `.html` 源文件，方便后续改字重渲。

## 一致性原则
同一份文档/汇报里的多张图，**沿用同一套配方和组件语言**（同一个 `--accent`/`--accent2`、同样的背景、同样的卡片/徽标/图标风格），这样即便每张图版式各不相同，也天然成系列、显专业。统一靠的是**配色 + 组件**，不是强行套同一个模板——所以放心按内容换版式，别为了「统一」牺牲表达。

## 给用户的可选项
渲染后可主动提示用户这些常见微调（都只需改 CSS 重渲）：
- 换主题色（青/紫/绿/橙，或暗黑科技底）；
- 收紧/放宽间距、调整图宽；
- 百分比类指标可加环形进度图（在 KPI 卡里嵌一个 conic-gradient 圆环）。
